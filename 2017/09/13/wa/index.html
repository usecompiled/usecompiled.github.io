Wa,Devstack

注意：本人13日在本文中写的关于暴力解决的方法（就是下一段落所说的这个办法）有一定的问题，当出现generate-subunit ……… Fail时，报错信息在exit_trap上面，如下图所示，大家可以针对具体报错信息来完成debug。

戳我看图！

如上图所示的问题，大家可以先复制该代码，之后重新运行这句代码，发现是etcd-v3.1.7-linux-amd64.tar.gz这个压缩包出现问题，无法打开，我们通过rm -rf 的方法，删除该tar包，重新执行.stack.sh即可。 其他错误也有一些权限报错，使用root权限（sudo）重新运行出错代码之后继续./stack.sh。 同一段代码，一千台机器运行可能会报一千种错，学生才疏学浅，并不能遇到也并不可能知道所以问题，网上的大神很多，诸位有问题多Google。

遇到任何错误，首先再执行一次./stack.sh 如果报相同的错，则google之，若报新错，重复以上，无法google出答案的，先执行./unstack.sh,再执行./stack.sh

generate-subunit: command not found解决办法：（依次在root账户下执行以下操作） # sudo apt-get install python-pip # sudo pip install –upgrade pip # sudo pip install -U os-testr

注意：某些过往版本可能有兼容性问题，把执行语句换为下面这句：

Force=yes ./stack.sh

第一次安装(失败):

环境：Ubuntu 16.04 直接使用Dev stack官网提示，安装失败。 此次安装没有注意error信息，大失误。

第二次安装(失败):

内容回去写 环境:Ubuntu 14.04 server 安装至一半 自己手贱ctrl+c了，之后继续运行 卡 Nova-api 错误 寻找答案无果，遂罢。

第三次安装(成功):

环境：Ubuntu 16.04.03 Desktop(VmwareWorkstation 12 虚拟机分配4g内存) devstack(pike): https://github.com/openstack-dev/devstack.git 注意安装前基本不需要多余步骤。 安装时多次报错，某些在网上找到解决办法，某些没有，没有解决办法的统统跳过，结果成功（目前成功：2/2）。 成功打印： This is your host IP address: ..0.* This is your host IPv6 address: ::1 Horizon is now available at http://**/dashboard—————–X Keystone is serving at http:///identity/ The default users are: admin and demo The password: *——————————————————Y 使用X处网址登陆管理页面 username为admin password为Y处所示。
