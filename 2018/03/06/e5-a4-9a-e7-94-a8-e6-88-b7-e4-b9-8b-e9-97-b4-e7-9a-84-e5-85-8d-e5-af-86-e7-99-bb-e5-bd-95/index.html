多用户之间的免密登录

参考资料如下：

  1. linux ssh 实现免密登录
  2. CentOS7安装Hadoop2.7完整步骤

环境： 两台CentOS7 mininal (slave1,slave2),一台CentOS7 GNOME桌面版(Master)， 1.CentOS默认没有启动ssh无密登录，去掉/etc/ssh/sshd_config其中2行的注释，每台服务器都要设置， #RSAAuthentication yes #PubkeyAuthentication yes 将这两句的#号去掉，即： RSAAuthentication yes PubkeyAuthentication yes

1、在三台机器终端都执行以下命令：ssh-keygen -t rsa

生成本机的公钥和私钥文件，过程中不断敲回车即可，ssh-keygen命令默认会将公钥放在/root/.ssh目录下

2、cd /root/.ssh，进入.ssh目录

此时.ssh目录下有2个文件：

id_rsa : 生成的私钥文件

id_rsa.pub ： 生成的公钥文件

3、cp id_rsa.pub authorized_keys

将公钥复制为authorized_keys文件，此时使用ssh连接本机就不需要输入密码了

authorized_keys:存放远程免密登录的公钥,主要通过这个文件记录多台机器的公钥。

4、ssh 本机hostname或者ip，本机登录本机

.ssh目录下新增了一个文件

know_hosts : 已知的主机公钥清单。

5.将Slave服务器的公钥合并至Master的authorized_keys文件中

ssh root@ cat ~/.ssh/id_rsa.pub>> authorized_keys

6、把Master服务器的authorized_keys、known_hosts复制到Slave服务器的/root/.ssh目录

这样所有服务器的authorized_keys文件里面都会有彼此的公钥，服务器的know_hosts会保存登录进来的主机清单

OK!
